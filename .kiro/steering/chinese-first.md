# 中文优先交流规范

## 核心原则

本项目是**双语导师系统**，专为中文用户设计，因此所有交流、文档和产出都必须遵循**中文优先**原则。

## 交流规范

### 1. 与用户交流
- **必须使用中文**与用户进行所有对话
- 思考过程、分析、建议、总结都用中文表达
- 项目状态汇报、进度说明使用中文
- 问题诊断和解决方案用中文描述

### 2. 文档编写
- 所有`.kiro`文件夹中的文档**必须以中文为主**
- `specs/`文件夹中的需求、设计、任务文档使用中文
- 用户手册、说明文档、README等面向用户的内容使用中文
- 技术文档的概念解释和描述使用中文

### 3. 代码注释
- 面向用户功能的注释**必须使用中文**
- 业务逻辑相关的注释使用中文
- 复杂算法和技术实现可以使用英文，但**必须提供中文注释**

## 允许使用英文的情况

### 1. 技术必需
- 编程语言关键字（`def`, `class`, `import`等）
- 代码变量名和函数名（遵循Python命名规范）
- 文件路径和目录名（`bilingual_tutor/core/engine.py`）
- 命令行指令（`pip install`, `python -m pytest`）

### 2. 国际标准
- HTTP状态码、API端点名称
- 数据库字段名（遵循英文命名规范）
- 配置文件中的技术参数名

### 3. 必需英文时的处理
```python
# 示例：技术代码 + 中文注释
class CoreLearningEngine:  # 核心学习引擎
    """
    中央学习引擎 - 协调所有系统组件的核心类
    负责管理学习会话、时间分配和组件通信
    """
    
    def allocate_study_time(self, total_minutes: int):  # 分配学习时间
        """
        分配学习时间到不同活动和语言
        
        参数:
            total_minutes: 总学习时间（分钟）
        
        返回:
            TimeAllocation: 时间分配结果
        """
```

## 文档结构规范

### 1. 标题和章节
```markdown
# 主标题（中文）
## 二级标题（中文）
### 功能说明（中文）

#### 技术实现 Technical Implementation
- 中文描述为主
- 英文术语用括号补充：数据库（Database）
```

### 2. 代码块说明
```markdown
### 安装依赖
```bash
# 安装Python依赖包
pip install -r requirements.txt

# 运行测试套件
python -m pytest tests/ -v
```

**说明**：上述命令用于设置开发环境和验证系统功能。
```

### 3. 技术概念解释
- 首次出现的英文技术术语需要中文解释
- 示例：Flask（Python Web框架）、SQLite（轻量级数据库）
- 后续可以直接使用英文术语

## 执行要求

所有AI助手在此项目中工作时：

1. **必须遵循中文优先原则**
2. **技术术语首次使用时提供中文解释**
3. **代码注释优先使用中文**
4. **用户交流完全使用中文**
5. **文档编写以中文为主体**
6. **每次开始新任务时必须重新查看本规范文件**

## 任务执行规范

### 上下文保持要求
- **每次开始执行新任务时，必须首先阅读本chinese-first.md文件**
- **确保在任务执行过程中始终遵循中文优先原则**
- **特别是在需要用户做关键决策时，必须使用中文交流**
- **不得因为任务切换而忽略中文优先原则**

### 关键决策交流
- **所有需要用户确认的选择都必须用中文描述**
- **技术方案的利弊分析必须用中文解释**
- **错误诊断和解决方案必须用中文说明**
- **进度汇报和状态更新必须用中文进行**

### 违规后果
如果AI助手在任务执行中违反中文优先原则：
- **可能导致用户误解关键信息**
- **可能造成错误的技术决策**
- **影响项目的用户友好性目标**
- **违背双语导师系统的核心理念**

这是双语导师系统的核心特征，体现了对中文用户的友好支持。
